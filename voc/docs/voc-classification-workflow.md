# VOC 분류 작업 방법 및 순서 로직

## 1) 목적
- 입력 Excel에서 `2026 list` 시트의 `반응_제품반응` 항목만 찾아,
- `반응_문의 / 반응_교환 / 반응_구해요 / 반응_정보 / 반응_기타` 5개로 재분류하고,
- 결과를 `output/` 경로에 새 파일로 저장한다.

## 2) 입력/출력 규칙
- 입력 파일: 원본 `.xlsx` (덮어쓰기 금지)
- 기준 시트: `2026 list`
- 대상 범위: row `5~220`
- 분류 대상 컬럼:
  - 주요 내용: col `7` (쓰기 대상)
  - 제목: col `10` (1차 분류 입력)
  - 링크: col `11` (VOC 이미지 매칭 키)
- 출력 파일: `output/<입력파일명>`

## 3) 하이브리드 분류 로직 (핵심)

### Step A. 규칙 기반 우선 분류
- 제목에 교환 키워드 포함 시 `반응_교환`
- 제목에 구해요 키워드 포함 시 `반응_구해요`
- 이 단계에서 분류되면 API 호출 없이 즉시 확정

### Step B. LLM 제목 분류
- Step A에서 미분류된 항목만 `GPT-4.1`으로 제목 분류
- 정상 응답이고 결과가 `반응_기타`가 아니면 확정

### Step C. 기타 결과일 때 본문 이미지 OCR 재분류
- Step B 결과가 `반응_기타`이고 링크 매칭 이미지가 있으면,
- VOC 시트(`BP_반응`, `PK_반응`)의 **본문 이미지**를 Vision 입력으로 보내 재분류
- 본문 이미지 기준: 카드 내 이미지 중 `row > 6` (헤더 이미지 제외)

### Step D. API 에러 시에만 모델 폴백
- `GPT-4.1` 호출이 HTTP/네트워크 에러일 때만 `GPT-4o` 폴백
- `반응_기타`라는 분류 결과 자체는 폴백 트리거가 아님

## 4) VOC 이미지 인덱싱 로직
- VOC 시트에서 row 2의 `Voice Of Customer` 텍스트 위치를 카드 시작 컬럼으로 인식
- 각 카드의 링크는 row 4, `card_start + 4` 컬럼에서 읽음
- 카드 범위 내 이미지 중 본문 이미지(`row > 6`)만 수집
- 최종 맵 구조: `link -> [image_bytes, ...]`

## 5) 재시도/호출 제어 정책
- API 재시도: 모델당 최대 3회
- 지수 백오프: `1s -> 2s -> 4s`
- 호출 간 고정 딜레이: `0.5s`
- 참고: 현재 구현은 순차 처리이며, 운영 시 병렬화가 필요하면 동시 2건 제한으로 확장 가능

## 6) 실행 순서
1. 환경변수 로드 (`GITHUB_TOKEN`)
2. 입력 workbook 로드
3. VOC 이미지 인덱스 생성 (`build_image_index`)
4. 대상 row 수집 (`read_target_rows`)
5. 각 row에 대해 하이브리드 분류 수행 (`classify_item`)
6. col 7 값 갱신
7. 분류 요약 출력
8. `output/`에 저장

## 7) 운영 실행 명령어


## 9) 장애 대응 메모
- `429 Too Many Requests` 발생 시:
  - 일시 대기 후 재실행
  - 동일 토큰/환경에서 연속 호출을 줄여 재시도
- 링크 매칭 이미지가 없으면 이미지 OCR 단계는 건너뛰고 제목 분류 결과를 사용
