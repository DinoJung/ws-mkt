# VOC 분류 작업 방법 및 수정 계획

## 1) 목적
- 입력 Excel에서 `2026 list` 시트의 `반응_제품반응` 항목만 찾는다.
- 각 항목을 `반응_문의 / 반응_교환 / 반응_구해요 / 반응_정보 / 반응_기타` 5개 중 하나로 재분류한다.
- 재분류가 끝나면 `col 11` 링크를 기준으로 workbook 안의 연결된 카드 영역도 찾아 같은 분류값으로 동기화한다.
- 결과는 원본을 덮어쓰지 않고 `output/` 경로에 새 파일로 저장한다.

## 2) 입력/출력 규칙
- 입력 파일: 원본 `.xlsx` (덮어쓰기 금지)
- 기준 시트: `2026 list`
- 대상 범위: row `5~220`
- 분류 관련 컬럼:
  - 주요 내용: col `7` (`2026 list` 최종 쓰기 대상)
  - 제목: col `10` (1차 분류 입력)
  - 링크: col `11` (본문 재확인 및 카드 매칭 키)
- 연결 카드 갱신 규칙:
  - `col 11` 링크와 같은 링크를 가진 카드가 workbook 내 다른 시트에 있으면 해당 카드의 `반응_제품반응`도 같은 최종 분류값으로 변경한다.
  - 동일 링크가 여러 카드에 연결되어 있으면 전부 갱신한다.
- 출력 파일: `output/<입력파일명>`

## 3) 하이브리드 분류 로직 (수정 후)

### Step A. 규칙 기반 우선 분류
- 제목에 교환 키워드 포함 시 `반응_교환`
- 제목에 구해요 키워드 포함 시 `반응_구해요`
- 이 단계에서 분류되면 API 호출 없이 즉시 확정

### Step B. LLM 제목 분류
- Step A에서 미분류된 항목만 제목 기준으로 1차 분류한다.
- 기본 모델은 현재와 동일하게 `GPT-4.1`을 사용한다.
- 정상 응답이고 결과가 `반응_기타`가 아니면 즉시 확정한다.

### Step C. `반응_기타`일 때 링크 본문 재분류
- Step B 결과가 `반응_기타`이면 `col 11` 링크 페이지를 다시 확인한다.
- 페이지 HTML에서 `.article.viewer` 또는 이에 준하는 본문 영역을 우선 찾는다.
- 본문 영역 안의 공지/배너/주의 문구는 제거한다.
  - 예: `회원간의 거래 분쟁에 대한 공론화 금지`
- 제거 후 남은 본문 텍스트를 정리해서 ask-codex 호환 LLM 또는 적절한 분류 스킬에 전달한다.
- 이 단계에서는 가능한 한 `반응_OO` 형태의 구체 카테고리로 재분류를 시도한다.
- 본문 추출 또는 재분류가 실패하면 다음 단계로 넘어간다.

### Step D. Workbook 이미지 OCR 재분류
- Step C에서도 확정되지 않았고 링크 매칭 이미지가 있으면 workbook 안 카드의 **본문 이미지**를 Vision 입력으로 보내 재분류한다.
- 기존 VOC 시트(`BP_반응`, `PK_반응`) 로직을 유지하되, 필요 시 카드 시트 범위를 일반화할 수 있게 설계한다.
- 본문 이미지 기준: 카드 내 이미지 중 `row > 6` (헤더 이미지 제외)

### Step E. API 에러 시에만 모델 폴백
- 텍스트/본문/이미지 분류에서 `GPT-4.1` 호출이 HTTP/네트워크 에러일 때만 `GPT-4o` 폴백을 사용한다.
- `반응_기타`라는 분류 결과 자체는 폴백 트리거가 아니다.

### Step F. 최종 결과 확정
- Step C 또는 Step D에서 구체 카테고리가 나오면 그 값을 최종 결과로 사용한다.
- 모든 재분류 단계 후에도 근거가 없으면 최종 결과는 `반응_기타`로 유지한다.

## 4) 링크/카드 인덱싱 로직

### 4-1. 링크 기반 대상 row 수집
- `2026 list`에서 row `5~220` 범위를 순회한다.
- col `7` 값이 `반응_제품반응`인 row만 대상에 포함한다.
- 각 대상 row마다 제목(col `10`)과 링크(col `11`)를 함께 수집한다.

### 4-2. 카드 이미지 인덱스
- VOC 카드 시트에서 row 2의 `Voice Of Customer` 텍스트 위치를 카드 시작 컬럼으로 인식한다.
- 각 카드의 링크는 row 4, `card_start + 4` 컬럼에서 읽는다.
- 카드 범위 내 이미지 중 본문 이미지(`row > 6`)만 수집한다.
- 최종 맵 구조: `link -> [image_bytes, ...]`

### 4-3. 링크 기반 카드 위치 인덱스
- 이미지 인덱스와 별도로 `link -> [card_location, ...]` 구조를 만든다.
- `card_location`에는 최소한 아래 정보가 필요하다.
  - 시트명
  - 카드 시작 컬럼
  - 링크 셀 위치
  - 분류값을 써야 하는 셀 위치
- 특정 시트명에 하드코딩하지 말고 카드 레이아웃 기준으로 찾는다.

## 5) 본문 추출 및 정제 정책
- 링크 HTML fetch는 분류 루프와 분리된 독립 함수로 구현한다.
- 본문 추출 순서는 다음과 같다.
  1. `.article.viewer` 또는 유사 selector 우선
  2. 해당 영역이 없으면 페이지 본문 fallback
  3. 그래도 실패하면 본문 재분류 단계 생략
- 본문 정제 시 제거 대상:
  - 공지 문구
  - 반복되는 안내문
  - 스크립트/스타일/숨김 영역
- 정제 후 텍스트는 공백과 줄바꿈을 정상화해서 분류 입력으로 사용한다.
- 공지 제거가 과도해서 본문이 비면 Step C는 실패로 보고 다음 단계로 이동한다.

## 6) 재시도/호출 제어 정책
- API 재시도: 모델당 최대 3회
- 지수 백오프: `1s -> 2s -> 4s`
- 호출 간 고정 딜레이: `0.5s`
- 링크 본문 fetch 실패는 전체 작업 실패로 보지 않고 해당 row만 다음 보조 단계로 넘긴다.
- 참고: 현재 구현은 순차 처리이며, 운영 시 병렬화가 필요하면 동시 2건 제한으로 확장 가능

## 7) 실행 순서
1. 환경변수 로드 (`GITHUB_TOKEN`)
2. 입력 workbook 로드
3. 카드 이미지 인덱스 생성 (`build_image_index`)
4. 링크 기반 카드 위치 인덱스 생성 (`build_card_index` 성격의 단계)
5. 대상 row 수집 (`read_target_rows`)
6. 각 row에 대해 규칙 기반 분류 수행
7. 규칙 미분류 건에 대해 제목 LLM 분류 수행
8. 결과가 `반응_기타`이면 링크 본문 fetch / 본문 추출 / 공지 제거 / 본문 재분류 수행
9. 필요 시 workbook 이미지 OCR 재분류 수행
10. 최종 분류값을 `2026 list` col `7`에 기록
11. 같은 링크를 가진 카드 영역의 `반응_제품반응`도 동일 값으로 기록
12. 분류 요약 출력
13. `output/`에 저장

## 8) 운영 실행 명령어

```bash
cd /home/ws-mkt/voc
set -a && . ./.env && set +a
python3 classify_voc.py "/path/to/input.xlsx"
```

## 10) 장애 대응 메모
- `429 Too Many Requests` 발생 시:
  - 일시 대기 후 재실행
  - 동일 토큰/환경에서 연속 호출을 줄여 재시도
- 링크 본문을 가져오지 못하면:
  - 본문 재분류 단계는 건너뛴다.
  - 가능한 경우 workbook 이미지 OCR을 계속 시도한다.
- 링크 매칭 이미지도 없으면 최종적으로 제목 분류 결과 또는 `반응_기타`를 사용한다.
- 동일 링크가 여러 카드에 매핑되면 일부만 갱신하지 말고 전체를 갱신한다.
